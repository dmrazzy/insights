## Column Relationships

- **`artifact_id` is the canonical join key** — shared with `artifacts_by_project_v1`, `sboms_v0`, and other artifact-level tables.
- **`artifact_source` is always "GITHUB"** — this table only contains GitHub repositories.
- **`(artifact_namespace, artifact_name)` is the human-readable repo identifier** — equivalent to `org/repo` on GitHub.
- **`artifact_url`** is derived from namespace and name, eg `https://github.com/{namespace}/{name}`.

## Query Rules

### Filtering Rules

- **Filter by `is_fork = false`** to exclude forked repositories from analysis — forks often inflate repo counts without representing original work.
- **Filter by `language`** to scope to a specific programming language.
- **Filter by `license_spdx_id`** for license compliance analysis.
- **No partition filtering** — FULL table, all predicates are post-scan.

### Aggregation Rules

- **`SUM(star_count)`, `SUM(fork_count)`** for aggregate popularity metrics across a set of repos.
- **`COUNT(*) WHERE is_fork = false`** for counting original (non-forked) repositories.
- **`COUNT(DISTINCT language)`** per project for technology diversity metrics.

### Join Rules

- **Join to `oso.artifacts_by_project_v1` on `artifact_id`** to attribute repositories to projects. This is the primary join pattern — `repositories_v0` has no project information on its own.
- **Join to `oso.sboms_v0` on `artifact_id = dependent_artifact_id`** to find what packages this repo depends on.
- **Cardinality**: one row per repository. Joining to `artifacts_by_project_v1` may produce multiple rows if the same repo is claimed by different project sources.

### Common Patterns

**Get repo metadata for an OSSD project:**
```sql
SELECT
  r.artifact_namespace,
  r.artifact_name,
  r.star_count,
  r.fork_count,
  r.language,
  r.license_spdx_id,
  r.is_fork
FROM oso.repositories_v0 AS r
INNER JOIN oso.artifacts_by_project_v1 AS a
  ON r.artifact_id = a.artifact_id
WHERE a.project_source = 'OSS_DIRECTORY'
  AND a.project_name = 'uniswap'
  AND a.artifact_source = 'GITHUB'
ORDER BY r.star_count DESC
```

**Top repositories by stars across all OSSD projects:**
```sql
SELECT
  a.project_name,
  r.artifact_namespace,
  r.artifact_name,
  r.star_count,
  r.fork_count,
  r.language
FROM oso.repositories_v0 AS r
INNER JOIN oso.artifacts_by_project_v1 AS a
  ON r.artifact_id = a.artifact_id
WHERE a.project_source = 'OSS_DIRECTORY'
  AND a.artifact_source = 'GITHUB'
  AND r.is_fork = false
ORDER BY r.star_count DESC
LIMIT 50
```

**Language distribution across an ecosystem:**
```sql
SELECT
  r.language,
  COUNT(*) AS repo_count,
  SUM(r.star_count) AS total_stars
FROM oso.repositories_v0 AS r
INNER JOIN oso.artifacts_by_project_v1 AS a
  ON r.artifact_id = a.artifact_id
INNER JOIN oso.projects_by_collection_v1 AS c
  ON a.project_id = c.project_id
WHERE c.collection_source = 'OSS_DIRECTORY'
  AND c.collection_name = 'ethereum'
  AND a.artifact_source = 'GITHUB'
  AND r.language IS NOT NULL
GROUP BY r.language
ORDER BY repo_count DESC
```

## Business Logic

This mart is a direct projection from `int_repositories__ossd`, which reads from `stg_ossd__current_repositories` — the staging layer over OSS Directory's curated repository list. The staging table parses GitHub repository URLs from OSSD YAML files using the `@parse_github_repository_artifact()` UDF and enriches them with metadata from GitHub's API (stars, forks, language, license, etc.).

Only repositories registered in OSS Directory appear in this table. Repositories discovered through other sources (OP Atlas, DeFiLlama) are not included — this is a curated, OSSD-only view of GitHub repos.

## Best Practices

- **Always join to `artifacts_by_project_v1`** to attribute repos to projects — `repositories_v0` has no project information on its own.
- **Filter `is_fork = false`** for original work analysis — forked repos are included but often represent copies, not original development.
- **Use `artifact_id` for joins**, not `artifact_name` — names can collide across organizations.
- **`star_count` and `fork_count` are point-in-time snapshots** — they reflect the last pipeline refresh, not real-time GitHub values.
- **`language` is GitHub's detected primary language** — repos with multiple languages only show the dominant one.
- **This table is OSSD-scoped** — it only contains repositories registered in OSS Directory. To check if a repo exists in other registries, query `artifacts_by_project_v1` directly.
