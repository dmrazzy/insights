## Column Relationships

- **`to_artifact_id` matches `project_id` in `projects_v1`** for OSSD-sourced funding (oss-funding repo and Gitcoin mapped projects). For unmapped Gitcoin recipients, it falls back to an address-based ID.
- **`from_artifact_id`** identifies the funder as an OSSD entity. For oss-funding data, this is `@oso_entity_id('OSS_DIRECTORY', 'oso', from_funder_name)`.
- **`event_type` splits Open Collective from grant data** ‚Äî CREDIT/DEBIT are Open Collective treasury flows; FUNDING_AWARDED is from oss-funding and Gitcoin.
- **`amount` is pre-aggregated to daily grain** ‚Äî multiple events on the same day for the same (source, type, from, to) tuple are summed.

## Query Rules

### Filtering Rules

- **Filter `event_type = 'FUNDING_AWARDED'`** for grant analysis. Without this, Open Collective CREDIT/DEBIT events dominate (3.85M of 4M rows) and represent general treasury flows, not grants.
- **Filter by `event_source`** to scope to a specific funder (eg `WHERE event_source = 'OCTANT'`).
- **No partition filtering** ‚Äî FULL table, all predicates are post-scan.

### Aggregation Rules

- **`SUM(amount)`** for total funding ‚Äî amounts are pre-normalized to USD.
- **`COUNT(DISTINCT to_artifact_id)`** for counting funded projects.
- **Always include `event_type` filter** in aggregations to avoid mixing grant awards with Open Collective treasury flows.

### Join Rules

- **Join to `oso.projects_v1` on `to_artifact_id = project_id`** to get project names. Use `WHERE project_source = 'OSS_DIRECTORY'` ‚Äî funding IDs are generated from OSSD project names.
- **Join to `oso.projects_by_collection_v1` on `to_artifact_id = project_id`** to group funded projects by ecosystem. Use `WHERE collection_source = 'OSS_DIRECTORY'`.
- **Join to `oso.artifacts_by_project_v1` on `to_artifact_id = project_id`** to get repos/contracts for funded projects.
- **Cardinality**: many rows per project (one per day √ó source √ó type). Joins to `projects_v1` do not fan out.

### Common Patterns

**Total FUNDING_AWARDED by source:**
```sql
SELECT
  event_source,
  COUNT(*) AS event_count,
  SUM(amount) AS total_usd,
  COUNT(DISTINCT to_artifact_id) AS projects_funded
FROM oso.int_events_daily__funding
WHERE event_type = 'FUNDING_AWARDED'
GROUP BY event_source
ORDER BY total_usd DESC
```

**Top funded OSSD projects:**
```sql
SELECT
  p.project_name,
  p.display_name,
  SUM(f.amount) AS total_funding_usd,
  COUNT(DISTINCT f.event_source) AS num_funders
FROM oso.int_events_daily__funding AS f
INNER JOIN oso.projects_v1 AS p
  ON f.to_artifact_id = p.project_id
WHERE p.project_source = 'OSS_DIRECTORY'
  AND f.event_type = 'FUNDING_AWARDED'
GROUP BY p.project_name, p.display_name
ORDER BY total_funding_usd DESC
LIMIT 30
```

**Octant funding by project:**
```sql
SELECT
  p.project_name,
  SUM(f.amount) AS octant_funding_usd
FROM oso.int_events_daily__funding AS f
INNER JOIN oso.projects_v1 AS p
  ON f.to_artifact_id = p.project_id
WHERE p.project_source = 'OSS_DIRECTORY'
  AND f.event_source = 'OCTANT'
GROUP BY p.project_name
ORDER BY octant_funding_usd DESC
```

## Business Logic

This model is a simple UNION ALL of three daily-aggregated funding sources:

1. **`int_events_daily__gitcoin_funding`** ‚Äî Gitcoin donations and matching, aggregated to daily grain with `from_artifact_id` set to the Gitcoin entity and `to_artifact_id` set to either the OSO project or a Gitcoin address fallback.
2. **`int_events_daily__ossd_funding`** ‚Äî oss-funding CSV data transformed from `stg_ossd__current_funding`. Maps `from_funder_name` to `event_source` values (OPTIMISM_RETROFUNDING, OCTANT, etc.) and resolves project/funder names to entity IDs via `@oso_entity_id`.
3. **`int_events_daily__open_collective_funding`** ‚Äî Open Collective expenses and deposits, incrementally processed.

No deduplication is applied at this layer ‚Äî the three upstream sources are assumed to be non-overlapping. All amounts are USD.

## Best Practices

- **Always filter `event_type = 'FUNDING_AWARDED'`** for grant analysis ‚Äî Open Collective CREDIT/DEBIT events are treasury flows, not grants.
- **Join to `projects_v1` with `project_source = 'OSS_DIRECTORY'`** ‚Äî funding IDs are OSSD-scoped.
- **Use `event_source` to identify the funder** ‚Äî not `from_artifact_id` (which is a hash).
- **Amounts are USD at time of award** ‚Äî no real-time price adjustments for crypto-denominated grants.
- **Data spans 2015‚Äì2026** ‚Äî earliest records are from Open Collective; most grant data starts from 2020+.
- **43 rows have null `bucket_day`** ‚Äî negligible but be aware in time-range queries.
